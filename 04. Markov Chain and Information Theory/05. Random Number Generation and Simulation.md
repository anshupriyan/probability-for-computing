# 05. Random Number Generation and Simulation (Section 11.1)

Sometimes, probability systems are too complex to solve with pen and paper. 

For instance, finding the exact expected waiting time in a massive airport queue with dozens of gates and varying flight arrival rates might require solving an impossible, multi-dimensional integral. 

When math becomes analytically intractable, we turn to **Simulation**.

---

## 1. The Core Analogy: Estimating Area by Throwing Darts

Imagine you draw a weird, irregular shape on a square board and want to calculate its area. 
* **The Analytical Way**: Try to write down a complex mathematical formula for the borders of the shape and integrate it. (Very hard!)
* **The Simulation Way**: 
  1. Put a blindfold on, and throw thousands of darts randomly at the board.
  2. Count how many darts land inside the shape versus how many land on the board in total.
  3. The proportion of darts that land inside the shape multiplied by the total area of the board gives an incredibly close estimate of the shape's area!

```
┌─────────────────────────┐
│  •    •     •      •    │
│    ┌─────────┐   •      │
│ •  │  •   •  │ •      • │
│    │ •     • │   •      │
│  • └─────────┘     •    │
│      •     •     •      │
└─────────────────────────┘
```
This is the foundational idea of **Monte Carlo Simulation**: we use random trials to approximate deterministic values.

---

## 2. The Monte Carlo Concept

Suppose we want to compute the expected value of some function of a random vector, $E[g(\boldsymbol{X})]$, where:
$$E[g(\boldsymbol{X})] = \iint \cdots \int g(x_1, \dots, x_n) f(x_1, \dots, x_n) \, dx_1 \cdots dx_n$$

If we cannot calculate this integral directly, we can estimate it using the following algorithm:

1. Generate $r$ independent, random vectors $\boldsymbol{X}^{(1)}, \boldsymbol{X}^{(2)}, \dots, \boldsymbol{X}^{(r)}$, each having the joint density function $f(x_1, \dots, x_n)$.
2. Calculate the function value for each vector: $Y^{(i)} = g(\boldsymbol{X}^{(i)})$ for $i = 1, \dots, r$.
3. By the **Strong Law of Large Numbers**, as the number of trials $r$ becomes large:
   $$\lim_{r \to \infty} \frac{Y^{(1)} + Y^{(2)} + \dots + Y^{(r)}}{r} = E[Y] = E[g(\boldsymbol{X})]$$
4. We can use the sample average $\bar{Y}$ as a highly accurate approximation of $E[g(\boldsymbol{X})]$.

## 3. Simulating Uniform $(0, 1)$ Random Variables

To simulate any random variable (discrete or continuous), we always start with the same basic building block: **Uniform $(0, 1)$ random variables**. 
If we can generate random numbers between 0 and 1, we can mathematically transform them into any other distribution (like Normal, Exponential, or Binomial).

### Physical Methods (True Randomness)
Historically, people generated random numbers using physical objects:
* Drawing slips of paper numbered $0$ to $9$ out of a hat with replacement.
* Using printed physical books of random numbers (like the famous RAND Corporation table of *A Million Random Digits*).

#### How to read a Random Number Table:
To construct a Uniform $(0, 1)$ random variable from a table:
1. Select a starting point (e.g., Row 1, Group 1) and decide on the precision $n$ (the number of digits you want).
2. Read $n$ consecutive digits from that starting point.
3. Place a decimal point in front of those digits.

* **Example**:
  - If we choose a precision of $n = 5$ digits and start at Row 1, Group 1, the digits are `04839`. The uniform random variable is $U = 0.04839$.
  - If we want a second number using the next group (Row 1, Group 2), the digits are `96423`, yielding $U_2 = 0.96423$.
  - If we only need 3-digit precision starting at Row 2, Group 3 (`46901`), we take the first 3 digits: `469`. The random variable is $U_0 = 0.469$.

These physical tables are slow to search and consume storage, which is why computers use algorithms instead.

## 4. Digital Methods: Pseudo-Random Number Generators (PRNGs)

Instead of using physical randomness, computers use deterministic formulas to generate sequences of numbers that *look* random. Because these sequences are completely determined by an initial value (the seed), they are called **Pseudo-Random Numbers**.

### The Linear Congruential Generator (LCG)
The most common technique to generate pseudo-random numbers is the **Linear Congruential Generator (LCG)**. 

To set up an LCG, we choose three positive integers:
1. **$a$ (Multiplier)**
2. **$c$ (Increment)**
3. **$m$ (Modulus)**
4. **$X_0$ (The Seed)**: The starting integer of the sequence.

We then recursively generate a sequence of integers $\{X_n\}$ using the formula:
$$X_{n+1} = (a X_n + c) \pmod m \quad \text{for } n \ge 0$$

* **How Modulo works**: The term $(a X_n + c) \pmod m$ means we divide $a X_n + c$ by $m$ and take the remainder.
* Since the remainder must be strictly less than the divisor, each generated integer $X_n$ lies in the set:
  $$X_n \in \{0, 1, 2, \dots, m-1\}$$
* To get a value between 0 and 1, we divide $X_n$ by the modulus $m$:
  $$U_n = \frac{X_n}{m}$$
  The sequence $\{U_n\}$ serves as our approximation of independent and identically distributed (i.i.d.) Uniform $(0, 1)$ random variables.

---

## 5. Walkthrough LCG Example

Let's build a simple LCG and generate the first few pseudo-random numbers.

* **Parameters**:
  - Modulus: $m = 8$
  - Multiplier: $a = 5$
  - Increment: $c = 1$
  - Seed: $X_0 = 3$

### Calculating the Sequence:

1. **Step 1 ($n=0$)**:
   $$X_1 = (a X_0 + c) \pmod m = (5 \cdot 3 + 1) \pmod 8 = 16 \pmod 8 = 0$$
   $$U_1 = \frac{X_1}{m} = \frac{0}{8} = 0.0$$

2. **Step 2 ($n=1$)**:
   $$X_2 = (a X_1 + c) \pmod m = (5 \cdot 0 + 1) \pmod 8 = 1 \pmod 8 = 1$$
   $$U_2 = \frac{X_2}{m} = \frac{1}{8} = 0.125$$

3. **Step 3 ($n=2$)**:
   $$X_3 = (a X_2 + c) \pmod m = (5 \cdot 1 + 1) \pmod 8 = 6 \pmod 8 = 6$$
   $$U_3 = \frac{X_3}{m} = \frac{6}{8} = 0.75$$

4. **Step 4 ($n=3$)**:
   $$X_4 = (a X_3 + c) \pmod m = (5 \cdot 6 + 1) \pmod 8 = 31 \pmod 8 = 7$$
   $$U_4 = \frac{X_4}{m} = \frac{7}{8} = 0.875$$

* **The Generated Pseudo-Random Sequence**: $\{0.0, \, 0.125, \, 0.75, \, 0.875\}$.

### ⚠️ Crucial Considerations:
* **The Period**: Since the sequence is deterministic, once it generates a state it has visited before, the sequence will repeat itself forever. The length of the sequence before it repeats is called the **period**.
* **Good Parameters**: To get a long period and good statistical properties (so the numbers look independent), developers choose massive numbers for $m$ (like $2^{31}-1$) and carefully tested prime values for $a$ and $c$.
