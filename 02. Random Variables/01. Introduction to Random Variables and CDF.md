# 01. Introduction to Random Variables and CDF (Section 2.1)

Welcome! If you are reading about probability theory, you might have noticed that sample spaces can get incredibly messy. If we flip ten coins, our outcomes are long strings of heads and tails like $HHTHTTTHTH$. If we measure how long a light bulb lasts, the outcomes are infinite time intervals. 

To make sense of this real-world chaos, mathematicians created a tool to act as a **translator**: the **Random Variable**.

---

## 1. The Core Analogy: The "Translator" (or "Scoring System")

Imagine you are playing a video game. The game does not care about the exact sequence of buttons you pressed, nor does it display your inputs on the high-score screen. Instead, it takes your complex actions (the outcome of your play) and translates them into a single, clean number: your **score**.

A **Random Variable** does the exact same thing:

> 💡 **Mental Model**: A Random Variable is not a "variable" in the traditional algebraic sense (like $x = 5$). It is actually a **function** (a rule) that maps each outcome in a sample space to a real number. 

```
[ Real-World Outcome ] ──( Random Variable / Translator )──> [ Clean Number ]
  e.g., (Heads, Tails)   ───────────────────────────────────>      1
```

Because the outcome of the experiment is random, the translated number is also random. We can therefore calculate the probability of obtaining different scores.

---

## 2. Walkthrough Examples

Let's look at how this mapping works in practice.

### Example A: The Sum of Two Dice (Example 2.1)
Suppose you roll two fair dice. The sample space $S$ contains 36 outcomes like $(1,1), (3,4),$ or $(6,6)$. 
Instead of tracking the individual faces, you are playing a game where only the **sum** of the faces matters. 

We define a random variable $X$ as:
$$X(\text{outcome}) = \text{Sum of the two faces}$$

Let's look at the mapping:
- If the outcome is $(1, 1)$, the translator outputs $X(1,1) = 2$.
- If the outcome is $(3, 4)$, the translator outputs $X(3,4) = 7$.
- If the outcome is $(4, 3)$, the translator outputs $X(4,3) = 7$.

Now, we can assign probabilities to these translated numbers:
* The only way to get a score of $2$ is the outcome $(1,1)$. Thus:
  $$P(X = 2) = P(\{(1,1)\}) = \frac{1}{36}$$
* There are six ways to get a score of $7$: $(1,6), (2,5), (3,4), (4,3), (5,2), (6,1)$. Thus:
  $$P(X = 7) = \frac{6}{36}$$

If we calculate the probabilities for all possible scores (from 2 to 12), they must sum to 1 because the dice *must* sum to some number in that range:
$$\sum_{n=2}^{12} P(X = n) = 1$$

### Example B: Waiting for Success (Example 2.3)
Imagine flipping a coin (probability $p$ of landing heads) repeatedly until you get your first head. We want to track the **number of flips required**.
We define a random variable $N$ as the number of flips.
- If the first flip is Heads ($H$), the experiment stops. Flips = 1.
  $$P(N = 1) = P(\{H\}) = p$$
- If the first flip is Tails and the second is Heads ($TH$), the experiment stops. Flips = 2.
  $$P(N = 2) = P(\{TH\}) = (1-p)p \quad \text{(due to independence)}$$
- In general, to stop at flip $n$, you must get $n-1$ tails followed by a head:
  $$P(N = n) = P(\{\underbrace{T, T, \dots, T}_{n-1 \text{ times}}, H\}) = (1-p)^{n-1}p$$

We can verify that our translator is mathematically sound by checking if the sum of all probabilities equals 1:
$$\sum_{n=1}^\infty P(N = n) = p \sum_{n=1}^\infty (1-p)^{n-1} = p \left[\frac{1}{1 - (1-p)}\right] = \frac{p}{p} = 1$$
*(Here we used the geometric series formula: $\sum_{k=0}^\infty q^k = \frac{1}{1-q}$ for $|q| < 1$.)*

## 3. Special Case: The Indicator Variable (Example 2.4)

Sometimes we only care about a binary question: **Did a specific event occur, yes or no?**

To represent this, we use an **Indicator Random Variable** (usually written as $I$). 
Think of it like a **sensor light**: it turns `1` (ON) if the event occurs, and `0` (OFF) if it does not.

> **Definition**: The indicator random variable $I$ for an event $E$ is defined by:
> $$I = \begin{cases} 
> 1, & \text{if } E \text{ occurs} \\
> 0, & \text{otherwise}
> \end{cases}$$

For example, if you test a battery and only care if it lasts at least 2 years:
- $I = 1$ if lifetime $\ge 2$ years.
- $I = 0$ if lifetime $< 2$ years.

## 4. Advanced Case: The Collector's Problem (Example 2.5)

Suppose you are collecting toys out of cereal boxes. There are $m$ different toys. In each box you buy, the probability of getting toy $i$ is $p_i$ (where $\sum_{i=1}^m p_i = 1$). 
How many boxes ($X$) do you need to buy until you have collected **every single toy** at least once?

Let's break down this concept step-by-step:
1. Finding the probability of getting *exactly* all toys on box $n$ ($P(X = n)$) directly is extremely hard.
2. Instead, we look at the probability that we **haven't finished** collecting all toys after $n$ boxes: $P(X > n)$.
3. Let $A_i$ be the event that **toy $i$ is missing** after $n$ boxes. 
   - If we haven't finished, it means at least one toy is missing. In set notation, this is a union of missing events:
     $$P(X > n) = P\left(\bigcup_{i=1}^m A_i\right)$$
4. By the Inclusion-Exclusion identity (which we learned in Unit 1):
   $$P\left(\bigcup_{i=1}^m A_i\right) = \sum_{i=1}^m P(A_i) - \sum_{i < j} P(A_i A_j) + \sum_{i < j < k} P(A_i A_j A_k) - \dots$$
5. Because each box is independent, the probability that toy $i$ is missing in all $n$ boxes is:
   $$P(A_i) = (1 - p_i)^n$$
   Similarly, the probability that both toy $i$ and toy $j$ are missing is:
   $$P(A_i A_j) = (1 - p_i - p_j)^n$$
6. Pushing these back into the equation gives us the probability of not finishing after $n$ trials:
   $$P(X > n) = \sum_{i=1}^m (1 - p_i)^n - \sum_{i < j} (1 - p_i - p_j)^n + \dots$$
7. Now, we use the logical bridge: the probability of finishing *exactly* at trial $n$ is the probability that we were not finished at $n-1$, minus the probability that we are still not finished at $n$:
   $$P(X = n) = P(X > n-1) - P(X > n)$$
8. Applying the algebraic identity $(1-a)^{n-1} - (1-a)^n = a(1-a)^{n-1}$:
   $$P(X = n) = \sum_{i=1}^m p_i (1-p_i)^{n-1} - \sum_{i < j} (p_i + p_j)(1 - p_i - p_j)^{n-1} + \dots$$

## 5. Cumulative Distribution Function (CDF)

Once we have mapped our outcomes to numbers, we want a unified tool to describe the distribution of these numbers.

> **Definition**: The **cumulative distribution function (CDF)** of a random variable $X$, denoted by $F(b)$, is:
> $$F(b) = P(X \le b) \quad \text{for } -\infty < b < \infty$$

### 💡 The Analogy: The "Progress Bar"
Think of the CDF as a **progress bar** loading from $0\%$ to $100\%$ as you move from left ($-\infty$) to right ($+\infty$). 
* At the far left ($-\infty$), the progress bar has not started: $F(-\infty) = 0$.
* As you move right, the bar can only fill up; it can never lose progress: $F(b)$ is **nondecreasing**.
* At the far right ($+\infty$), the progress bar is fully loaded: $F(\infty) = 1$.

```
Probability Load:  0% |═══════════════════▒▒▒▒▒▒▒▒▒▒▒▒| 100%
                      -∞                  b                  +∞
                      F(-∞)=0             F(b) = P(X ≤ b)    F(∞)=1
```

### Finding Interval Probabilities
If you want to find the probability that $X$ falls between $a$ and $b$ (i.e. $P(a < X \le b)$):
$$P(a < X \le b) = F(b) - F(a)$$
* **Intuition**: This is the progress accumulated up to $b$, minus the progress we had already accumulated up to $a$.
